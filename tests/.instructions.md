# Test Development Quick Reference

Fast guidance for writing tests, page objects, and step definitions in this project.

## File Organization

```
tests/
├── features/          → .feature files with Gherkin scenarios
├── steps/             → Step definition implementations (*.steps.ts)
├── pages/             → Page Object Models (POM)
├── hooks/             → Before/After lifecycle hooks
├── config/            → Constants, browser config, test data
├── utils/             → Logging, browser management utilities
└── fixtures/          → Static test data (JSON files)
```

## Writing Step Definitions

### Basic Pattern

```typescript
import { Given, When, Then } from "@cucumber/cucumber";
import { expect } from "@playwright/test";
import { CustomWorld } from "../support/CustomWorld";
import { logger } from "../utils/Logger";

Given(
  "I navigate to the SauceDemo login page",
  async function (this: CustomWorld) {
    logger.info("Navigating to login page");
    await this.loginPage.goto();
  },
);

When(
  "I enter username {string}",
  async function (this: CustomWorld, username: string) {
    await this.loginPage.fillUsername(username);
  },
);

Then("I should see an error message", async function (this: CustomWorld) {
  const errorVisible = await this.loginPage.isErrorMessageDisplayed();
  expect(errorVisible).toBe(true);
});
```

### Accessing Page Objects

All page objects are available via `CustomWorld` context (`this`):

- `this.loginPage` - LoginPage instance
- `this.inventoryPage` - InventoryPage instance
- `this.page` - Raw Playwright Page object

### Logging

```typescript
import { logger } from "../utils/Logger";

logger.info("User message"); // Info level
logger.debug("Debug details"); // Debug level
logger.warn("Warning message"); // Warning level
logger.error("Error occurred"); // Error level
```

## Creating Page Objects

### Template

```typescript
import { Page } from "@playwright/test";
import { BasePage } from "./BasePage";
import { logger } from "../utils/Logger";

export class CheckoutPage extends BasePage {
  // Locators as protected properties
  protected readonly continueButton = this.page.locator(
    'button:has-text("Continue")',
  );
  protected readonly priceTotal = this.page.locator(
    '[data-test="total-price"]',
  );

  constructor(page: Page) {
    super(page);
  }

  // Public methods for interactions
  async clickContinue(): Promise<void> {
    logger.info("Clicking continue button");
    await this.continueButton.click();
  }

  async getPriceTotal(): Promise<string> {
    return (await this.priceTotal.textContent()) || "";
  }

  async isCheckoutPageVisible(): Promise<boolean> {
    return await this.page.locator('[data-test="checkout"]').isVisible();
  }
}
```

### Best Practices

- **Extend BasePage** for common functionality (navigation, waits, etc.)
- **Locators as properties** - Define selectors at the top
- **Methods for interactions** - Hide details from step definitions
- **Meaningful names** - `clickLoginButton()`, not `click_button_1()`
- **Encapsulate waits** - Don't expose waitForNavigation in step definitions
- **Add logging** - Log important actions for debugging

## Writing Feature Files

### Scenario Template

```gherkin
Feature: User Registration
  As a new user
  I want to register for an account
  So that I can access the platform

  @smoke @registration
  Scenario: Successful registration with valid data
    Given I navigate to the registration page
    When I enter email "user@example.com"
    And I enter password "SecurePass123"
    And I click the register button
    Then I should be redirected to the dashboard
    And I should see welcome message

  @regression
  Scenario: Registration fails with existing email
    Given I navigate to the registration page
    When I enter email "existing@example.com"
    And I click the register button
    Then I should see error "Email already registered"
```

### Tag Conventions

- `@smoke` - Critical path tests (run first)
- `@regression` - Full test suite validation
- `@feature-name` - Group tests by feature
- `@pending` - Tests not yet implemented

## Hooks Configuration

### CustomWorld Class

```typescript
import { World, IWorldOptions } from "@cucumber/cucumber";
import { Page } from "@playwright/test";
import { LoginPage } from "../pages/LoginPage";
import { InventoryPage } from "../pages/InventoryPage";

export class CustomWorld extends World {
  public page!: Page;
  public loginPage!: LoginPage;
  public inventoryPage!: InventoryPage;

  constructor(options: IWorldOptions) {
    super(options);
  }
}
```

### In hooks.ts

```typescript
import {
  Before,
  After,
  setWorldConstructor,
  ITestCaseHookParameter,
} from "@cucumber/cucumber";
import { CustomWorld } from "./CustomWorld";
import { BrowserManager } from "../utils/BrowserManager";
import { LoginPage } from "../pages/LoginPage";
import { InventoryPage } from "../pages/InventoryPage";

// Register CustomWorld with Cucumber
setWorldConstructor(CustomWorld);

Before(async function (this: CustomWorld, scenario: ITestCaseHookParameter) {
  // Initialize browser and pages
  await BrowserManager.getInstance().launchBrowser();
  this.page = await BrowserManager.getInstance().createNewPage();

  // Attach page objects to World instance
  this.loginPage = new LoginPage(this.page);
  this.inventoryPage = new InventoryPage(this.page);
});

After(async function (this: CustomWorld) {
  // Cleanup
  await BrowserManager.getInstance().closeBrowser();
});
```

## Common Assertions

```typescript
import { expect } from "@playwright/test";

// Text assertions
expect(text).toBe("exact match");
expect(text).toContain("substring");
expect(text).toMatch(/regex/);

// Visibility
expect(await element.isVisible()).toBe(true);

// Element state
expect(await input.inputValue()).toBe("value");
expect(await element.isEnabled()).toBe(true);
expect(await element.isChecked()).toBe(true);

// List assertions
expect(items.length).toBe(3);
expect(items).toContain("item");
```

## Running Tests

```bash
# All tests
yarn test

# Specific tag
yarn test:tags @smoke
yarn test:tags @registration

# Multiple tags (AND)
yarn test:tags "@smoke and @login"

# Headed browser (debugging)
yarn test:headed

# With debugger
yarn test:debug
```

## Debugging Tips

1. **Use headed mode**: `yarn test:headed` to see browser
2. **Add logging**: Use `logger.info()` to trace execution
3. **Increase timeouts**: Update `.env` for slow environments
4. **Inspect elements**: Use `await hooksContext.page.pause()` to inspect DOM
5. **Print values**: Use `console.log()` in step definitions
6. **Check selectors**: Verify locators match current DOM state

## Common Patterns

### Wait for Element

```typescript
await hooksContext.page.waitForSelector('[data-test="my-element"]');
```

### Wait for Navigation

```typescript
await Promise.all([
  hooksContext.page.waitForNavigation(),
  hooksContext.loginPage!.clickLoginButton(),
]);
```

### Handle Dynamic Content

```typescript
const count = await hooksContext.page.locator('[data-test="item"]').count();
expect(count).toBeGreaterThan(0);
```

### Fill Form Field with Clear

```typescript
const input = hooksContext.page.locator('[data-test="username"]');
await input.fill(""); // Clear first
await input.fill("new_value");
```

## Naming Conventions

| Type            | Convention             | Example                         |
| --------------- | ---------------------- | ------------------------------- |
| Feature files   | kebab-case             | `user-registration.feature`     |
| Step files      | kebab-case + .steps.ts | `registration.steps.ts`         |
| Page classes    | PascalCase             | `RegistrationPage`              |
| Methods         | camelCase              | `clickRegisterButton()`         |
| Private methods | \_leadingUnderscore    | `_validateEmail()`              |
| Constants       | UPPER_SNAKE_CASE       | `MAX_RETRY_COUNT`               |
| Test data       | descriptive            | `validEmail`, `invalidPassword` |

## File Checklist for New Test Feature

- [ ] Feature file created in `tests/features/`
- [ ] Page object created in `tests/pages/` (if new page)
- [ ] Step definitions in `tests/steps/feature-name.steps.ts`
- [ ] Hooks updated in `tests/support/hooks.ts` (if needed)
- [ ] CustomWorld updated in `tests/support/CustomWorld.ts` (new page properties)
- [ ] Scenarios tagged appropriately (`@smoke`, `@regression`)
- [ ] Step definitions match feature file text exactly
- [ ] Step definitions use `async function (this: CustomWorld)` signature
- [ ] Assertions use `@playwright/test` expect
- [ ] Logging added for key actions
- [ ] Tests run successfully: `yarn test:headed`
